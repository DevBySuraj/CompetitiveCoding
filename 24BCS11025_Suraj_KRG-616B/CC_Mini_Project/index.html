    ⬡ How the Min-Heap Engine Works

    What is a Min-Heap?

    A complete binary tree where each parent node has a value ≤ its children. The minimum element (highest priority task) always sits at the root — giving us O(1) peek.

    Time Complexities

    • insert() — O(log n) via heapify-up
    • extractMin() — O(log n) via heapify-down
    • getMin() — O(1), always heap[0]
    • delete(i) — O(log n) after splice + rebuild

    Why not a sorted array?

    A sorted array costs O(n) per insertion (shifting). A heap costs only O(log n) — crucial when you have hundreds of tasks being added/removed rapidly.

    Implementation

    Stored as a flat array. For index i: left child = 2i+1, right = 2i+2, parent = ⌊(i−1)/2⌋. No pointers needed.
